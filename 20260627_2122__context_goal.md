### 🎯 Project Summary: Yamal-v1

**Core Mission:**
To build a **minimal, functional drag-and-drop workflow builder** that exports standard YAML configuration, serving as a foundational tool for developers.

### 🏗️ Key Objectives

1.  **Visual Workflow Builder**: Create an interactive canvas where users can drag and drop nodes (e.g., "HTTP Request", "Send Email") and connect them to define a workflow's logic.
2.  **YAML Export Engine**: Implement a backend function that parses the visual graph (nodes and connections) and generates valid, clean YAML output that represents the workflow.
3.  **Developer-First UX**: Build a simple, lightweight interface focused on speed and ease of use, prioritizing a "dead-simple" experience over enterprise-level complexity.
4.  **Framework Choice**: Use **Astro** and **React** (via React Flow) for a fast, static-first frontend with an interactive component "island."
5.  **Infrastructure**: Keep hosting costs minimal by leveraging Astro's static-site generation capabilities, as outlined in the hackathon's theme of building an "Operating Layer."
6.  **Deadline**: Complete a working prototype (MVP) by the submission deadline.

This project aims to sit in the sweet spot between overly complex tools and code-heavy solutions by providing a visual way to generate YAML without the bloat.
